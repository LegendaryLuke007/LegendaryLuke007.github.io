# Honey Pot Lab

---

## SUMMARY:

This is my second venture into working with Honey Pots (Post AWS T-Pot project). What I was originally planning to do was Creating a personal home network SOC... But due to the size/volume/intrigue of my network... I felt like it is worth to be a little more spicy and instead do a locally hosted Honey-Pot.

I Bought a decent Home-Lab computer (32 GB of Ram, 1 TB SSD, Core i7) so that I could have a enough room to have multiple VM projects set up in parallel. So, I plan on setting up a VM with adequate security measures that is open to the Internet and collects as much data as possible. The log data will be stored locally on my PC (some more technical complexity to work through), and I want to set up automations to keep specific action logs in there own folders. Also to have a message-app based system that would let me know if it records more interesting information (such as a Malware Capture, or actual security break ins).

This project will be quite interesting, especially since this Honey Pot could be super fun just because I will be able to get a constant flow of information in relation to current CVEs attempted, what are the most popular Exploit Vectors, etc. I ended up turning off my AWS Instance running T-Pot because I really have no good reason to burn $20 other then to look at cool graphs... This way I can just get away with my electrical bill payment per month (estimated to be around $5 with this computer running 247) and no continuous costs!
